# 8.6 Cancel Async Tasks After a Period of Time

## Description

Time-based cancellation is a common pattern in asynchronous programming where you want to automatically cancel operations after a specific duration. This is useful for implementing timeouts, preventing operations from running indefinitely, and ensuring responsive applications. C# provides several ways to implement time-based cancellation using `CancellationTokenSource.CancelAfter()` and `Task.Delay()`.

## Key Concepts

- **Timeout cancellation**: Automatically cancel operations after a specified time
- **CancelAfter()**: Method to schedule cancellation after a time period
- **Task.Delay()**: Used with cancellation tokens for timeouts
- **Timeout handling**: Graceful handling of timeouts in async operations

## Code Examples

### Basic Timeout Cancellation

```csharp
using System;
using System.Threading;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Basic Timeout Cancellation Demo");
        
        await BasicTimeoutDemo();
    }
    
    static async Task BasicTimeoutDemo()
    {
        using var cts = new CancellationTokenSource();
        
        // Cancel after 3 seconds
        cts.CancelAfter(TimeSpan.FromSeconds(3));
        
        try
        {
            Console.WriteLine("Starting long-running operation...");
            await LongRunningOperationAsync(cts.Token);
            Console.WriteLine("Operation completed successfully!");
        }
        catch (OperationCanceledException)
        {
            Console.WriteLine("Operation was cancelled due to timeout.");
        }
    }
    
    static async Task LongRunningOperationAsync(CancellationToken cancellationToken)
    {
        Console.WriteLine("Long operation started...");
        
        for (int i = 0; i < 10; i++)
        {
            cancellationToken.ThrowIfCancellationRequested();
            
            Console.WriteLine($"Working... {i + 1}/10");
            await Task.Delay(1000, cancellationToken);
        }
        
        Console.WriteLine("Long operation completed!");
    }
}
```

```

Working... 3/10
Operation was cancelled due to timeout.
```

```csharp
        // Strategy 1: Fixed timeout
        Console.WriteLine("=== Strategy 1: Fixed Timeout ===");

        Console.WriteLine("\n=== Strategy 2: Variable Timeout ===");

        Console.WriteLine("\n=== Strategy 3: Per-Operation Timeout ===");
        await PerOperationTimeoutDemo();

    static async Task FixedTimeoutDemo()
    {
        using var cts = new CancellationTokenSource();
        
        // Fixed 2-second timeout
        cts.CancelAfter(2000);
        
        try
        {
            await SimulateWorkAsync("Fixed Timeout", 5, cts.Token);

    {

            new { Name = "Quick Operation", Timeout = 1000, Duration = 500 },

            {
                Console.WriteLine($"{op.Name} was cancelled due to timeout.");
            }

        {
            "Database Query",
            "File Processing",
            "Network Request",
            "Data Validation"

            try
            {
                Console.WriteLine($"Starting {operation} (timeout: {timeout}ms)...");
                await SimulateWorkAsync(operation, 2, cts.Token);
                Console.WriteLine($"{operation} completed successfully!");
            }
            catch (OperationCanceledException)
            {
                Console.WriteLine($"{operation} was cancelled due to timeout.");
            }
        }
    }

        {
            "Database Query" => 2000,
            "File Processing" => 5000,
            "Network Request" => 10000,
            "Data Validation" => 1000,
            _ => 3000
        };
    }
    
    static async Task SimulateWorkAsync(string operationName, int seconds, CancellationToken cancellationToken)
    {
        Console.WriteLine($"{operationName}: Started");
        
        for (int i = 0; i < seconds; i++)
        {
            cancellationToken.ThrowIfCancellationRequested();
            
            Console.WriteLine($"{operationName}: Working... {i + 1}/{seconds}");
            await Task.Delay(1000, cancellationToken);
        }
        
        Console.WriteLine($"{operationName}: Completed");
    }
}
```

### Timeout with Retry Logic

```csharp
using System;
using System.Threading;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Timeout with Retry Logic Demo");
        
        await TimeoutWithRetryDemo();
    }
    
    static async Task TimeoutWithRetryDemo()
    {
        const int maxRetries = 3;
        const int timeoutMs = 2000;
        
        for (int attempt = 1; attempt <= maxRetries; attempt++)
        {
            Console.WriteLine($"\nAttempt {attempt}/{maxRetries}");
            
            using var cts = new CancellationTokenSource();
            cts.CancelAfter(timeoutMs);
            
            try
            {
                await UnreliableOperationAsync(cts.Token);
                Console.WriteLine($"Operation succeeded on attempt {attempt}!");
                break; // Success, exit retry loop
            }
            catch (OperationCanceledException)
            {
                Console.WriteLine($"Attempt {attempt} timed out after {timeoutMs}ms.");
                
                if (attempt == maxRetries)
                {
                    Console.WriteLine("All retry attempts failed due to timeout.");
                }
                else
                {
                    Console.WriteLine("Retrying...");
                    await Task.Delay(500); // Brief delay before retry
                }
            }
        }
    }
    
    static async Task UnreliableOperationAsync(CancellationToken cancellationToken)
    {
        Console.WriteLine("Starting unreliable operation...");
        
        // Simulate an operation that sometimes takes too long
        var random = new Random();
        int delay = random.Next(1000, 4000); // 1-4 seconds
        
        Console.WriteLine($"Operation will take approximately {delay}ms");
        
        for (int i = 0; i < delay / 500; i++)
        {
            cancellationToken.ThrowIfCancellationRequested();
            await Task.Delay(500, cancellationToken);
        }
        
        Console.WriteLine("Unreliable operation completed!");
    }
}
```

### Timeout with Fallback Operations

```csharp
using System;
using System.Threading;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Timeout with Fallback Operations Demo");
        
        await TimeoutWithFallbackDemo();
    }
    
    static async Task TimeoutWithFallbackDemo()
    {
        // Try primary operation with timeout
        Console.WriteLine("Trying primary operation...");
        string result = await TryOperationWithTimeoutAsync(
            PrimaryOperationAsync,
            "Primary",
            TimeSpan.FromSeconds(2)
        );
        
        Console.WriteLine($"Final result: {result}");
    }
    
    static async Task<string> TryOperationWithTimeoutAsync(
        Func<CancellationToken, Task<string>> operation,
        string operationName,
        TimeSpan timeout)
    {
        using var cts = new CancellationTokenSource();
        cts.CancelAfter(timeout);
        
        try
        {
            return await operation(cts.Token);
        }
        catch (OperationCanceledException)
        {
            Console.WriteLine($"{operationName} operation timed out, trying fallback...");
            return await FallbackOperationAsync();
        }
    }
    
    static async Task<string> PrimaryOperationAsync(CancellationToken cancellationToken)
    {
        Console.WriteLine("Primary operation started...");
        
        // Simulate a slow operation
        for (int i = 0; i < 5; i++)
        {
            cancellationToken.ThrowIfCancellationRequested();
            Console.WriteLine($"Primary: Working... {i + 1}/5");
            await Task.Delay(1000, cancellationToken);
        }
        
        Console.WriteLine("Primary operation completed!");
        return "Primary operation result";
    }
    
    static async Task<string> FallbackOperationAsync()
    {
        Console.WriteLine("Fallback operation started...");
        
        // Simulate a faster fallback operation
        await Task.Delay(500);
        
        Console.WriteLine("Fallback operation completed!");
        return "Fallback operation result";
    }
}
```

**Expected Output:**
```

Trying primary operation...
Primary operation started...
Primary: Working... 1/5
Primary: Working... 2/5
Primary operation timed out, trying fallback...
Fallback operation started...

Final result: Fallback operation result
```

### Advanced Timeout Patterns

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using System.Threading;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Advanced Timeout Patterns Demo");
        
        await AdvancedTimeoutPatterns();
    }
    
    static async Task AdvancedTimeoutPatterns()
    {
        // Pattern 1: Timeout with progress reporting
        Console.WriteLine("=== Pattern 1: Timeout with Progress ===");
        await TimeoutWithProgress();
        
        Console.WriteLine("\n=== Pattern 2: Cascading Timeouts ===");
        await CascadingTimeouts();
        
        Console.WriteLine("\n=== Pattern 3: Adaptive Timeouts ===");
        await AdaptiveTimeouts();
    }
    
    static async Task TimeoutWithProgress()
    {
        using var cts = new CancellationTokenSource();
        cts.CancelAfter(TimeSpan.FromSeconds(3));
        
        try
        {
            await OperationWithProgressAsync(cts.Token);
            Console.WriteLine("Operation completed successfully!");
        }
        catch (OperationCanceledException)
        {
            Console.WriteLine("Operation was cancelled due to timeout.");
        }
    }
    
    static async Task OperationWithProgressAsync(CancellationToken cancellationToken)
    {
        Console.WriteLine("Starting operation with progress reporting...");
        
        for (int i = 0; i < 10; i++)
        {
            cancellationToken.ThrowIfCancellationRequested();
            
            int progress = (i + 1) * 10;
            Console.WriteLine($"Progress: {progress}%");
            
            await Task.Delay(500, cancellationToken);
        }
        
        Console.WriteLine("Operation completed!");
    }
    
    static async Task CascadingTimeouts()
    {
        // Outer timeout: 5 seconds
        using var outerCts = new CancellationTokenSource();
        outerCts.CancelAfter(TimeSpan.FromSeconds(5));
        
        try
        {
            await CascadingOperationAsync(outerCts.Token);
            Console.WriteLine("Cascading operation completed!");
        }
        catch (OperationCanceledException)
        {
            Console.WriteLine("Cascading operation was cancelled.");
        }
    }
    
    static async Task CascadingOperationAsync(CancellationToken outerToken)
    {
        Console.WriteLine("Starting cascading operation...");
        
        // Inner timeout: 2 seconds
        using var innerCts = CancellationTokenSource.CreateLinkedTokenSource(outerToken);
        innerCts.CancelAfter(TimeSpan.FromSeconds(2));
        
        try
        {
            await InnerOperationAsync(innerCts.Token);
            Console.WriteLine("Inner operation completed!");
        }
        catch (OperationCanceledException) when (innerCts.Token.IsCancellationRequested && !outerToken.IsCancellationRequested)
        {
            Console.WriteLine("Inner operation timed out, trying alternative...");
            await AlternativeOperationAsync(outerToken);
        }
    }
    
    static async Task InnerOperationAsync(CancellationToken cancellationToken)
    {
        Console.WriteLine("Inner operation started...");
        await Task.Delay(3000, cancellationToken); // This will timeout
        Console.WriteLine("Inner operation completed!");
    }
    
    static async Task AlternativeOperationAsync(CancellationToken cancellationToken)
    {
        Console.WriteLine("Alternative operation started...");
        await Task.Delay(1000, cancellationToken);
        Console.WriteLine("Alternative operation completed!");
    }
    
    static async Task AdaptiveTimeouts()
    {
        var timeouts = new[] { 1000, 2000, 4000 }; // Increasing timeouts
        
        foreach (var timeout in timeouts)
        {
            Console.WriteLine($"Trying with {timeout}ms timeout...");
            
            using var cts = new CancellationTokenSource();
            cts.CancelAfter(timeout);
            
            try
            {
                await AdaptiveOperationAsync(cts.Token);
                Console.WriteLine($"Operation succeeded with {timeout}ms timeout!");
                break;
            }
            catch (OperationCanceledException)
            {
                Console.WriteLine($"Operation failed with {timeout}ms timeout.");
            }
        }
    }
    
    static async Task AdaptiveOperationAsync(CancellationToken cancellationToken)
    {
        Console.WriteLine("Adaptive operation started...");
        
        // Simulate variable processing time
        var random = new Random();
        int processingTime = random.Next(500, 3000);
        
        Console.WriteLine($"Processing will take {processingTime}ms");
        await Task.Delay(processingTime, cancellationToken);
        
        Console.WriteLine("Adaptive operation completed!");
    }
}
```

## Best Practices

1. **Set appropriate timeouts**: Consider the operation type and user expectations
2. **Use meaningful timeout values**: Don't use arbitrary timeouts
3. **Handle timeouts gracefully**: Provide fallback mechanisms when possible
4. **Implement retry logic**: For transient failures
5. **Report progress**: For long-running operations
6. **Use linked tokens**: For cascading timeouts
7. **Test timeout scenarios**: Ensure your code handles timeouts correctly
8. **Consider user experience**: Balance between responsiveness and completion
