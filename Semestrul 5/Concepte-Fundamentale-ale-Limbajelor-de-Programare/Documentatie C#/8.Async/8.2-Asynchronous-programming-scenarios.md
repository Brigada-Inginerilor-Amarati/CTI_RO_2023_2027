# 8.2 Asynchronous Programming Scenarios

## Description

This topic covers various real-world scenarios where asynchronous programming is beneficial. Understanding when and how to apply async/await patterns helps you write more responsive and efficient applications. Common scenarios include I/O operations, web requests, database calls, and UI responsiveness.

## Key Scenarios

1. **I/O Operations**: File reading/writing, network requests
2. **Database Operations**: Querying and updating data
3. **Web API Calls**: HTTP requests and responses
4. **UI Responsiveness**: Keeping user interface responsive
5. **Parallel Processing**: Multiple independent operations

## Code Examples

### Scenario 1: File I/O Operations

```csharp
using System;
using System.IO;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("File I/O Operations Demo");
        
        // Write to file asynchronously
        await WriteToFileAsync("sample.txt", "Hello, Async World!");
        
        // Read from file asynchronously
        string content = await ReadFromFileAsync("sample.txt");
        Console.WriteLine($"File content: {content}");
        
        // Multiple file operations
        await ProcessMultipleFilesAsync();
    }
    
    static async Task WriteToFileAsync(string fileName, string content)
    {
        Console.WriteLine($"Writing to {fileName}...");
        await File.WriteAllTextAsync(fileName, content);
        Console.WriteLine($"Successfully wrote to {fileName}");
    }
    
    static async Task<string> ReadFromFileAsync(string fileName)
    {
        Console.WriteLine($"Reading from {fileName}...");
        string content = await File.ReadAllTextAsync(fileName);
        Console.WriteLine($"Successfully read from {fileName}");
        return content;
    }
    
    static async Task ProcessMultipleFilesAsync()
    {
        var tasks = new[]
        {
            WriteToFileAsync("file1.txt", "Content 1"),
            WriteToFileAsync("file2.txt", "Content 2"),
            WriteToFileAsync("file3.txt", "Content 3")
        };
        
        await Task.WhenAll(tasks);
        Console.WriteLine("All files processed successfully!");
    }
}
```

**Expected Output:**

### Scenario 2: Web API Calls

```csharp
using System;
using System.Net.Http;
using System.Threading.Tasks;

class Program
{
    private static readonly HttpClient httpClient = new HttpClient();
    
    static async Task Main(string[] args)
    {
        Console.WriteLine("Web API Calls Demo");
        
        try
        {
            // Single API call
            string data = await FetchDataFromApiAsync("https://jsonplaceholder.typicode.com/posts/1");
            Console.WriteLine($"Single API Response: {data.Substring(0, 50)}...");
            
            // Multiple API calls concurrently
            await FetchMultipleDataAsync();
        }
        catch (Exception ex)
        {
            Console.WriteLine($"Error: {ex.Message}");
        }
    }
    
    static async Task<string> FetchDataFromApiAsync(string url)
    {
        Console.WriteLine($"Fetching data from {url}...");
        HttpResponseMessage response = await httpClient.GetAsync(url);
        response.EnsureSuccessStatusCode();
        string content = await response.Content.ReadAsStringAsync();
        Console.WriteLine("API call completed successfully.");
        return content;
    }
    
    static async Task FetchMultipleDataAsync()
    {
        Console.WriteLine("Fetching multiple data sources...");
        
        var tasks = new[]
        {
            FetchDataFromApiAsync("https://jsonplaceholder.typicode.com/posts/1"),
            FetchDataFromApiAsync("https://jsonplaceholder.typicode.com/posts/2"),
            FetchDataFromApiAsync("https://jsonplaceholder.typicode.com/posts/3")
        };
        
        var results = await Task.WhenAll(tasks);
        
        Console.WriteLine($"Fetched {results.Length} data sources successfully!");
        for (int i = 0; i < results.Length; i++)
        {
            Console.WriteLine($"Post {i + 1}: {results[i].Substring(0, 30)}...");
        }
    }
}
```

### Scenario 3: Database Simulation

```csharp
using System;
using System.Collections.Generic;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Database Operations Demo");
        
        // Simulate database operations
        await DatabaseOperationsDemo();
    }
    
    static async Task DatabaseOperationsDemo()
    {
        // Insert data
        Console.WriteLine("Inserting user data...");
        int userId = await InsertUserAsync("John Doe", "john@example.com");
        Console.WriteLine($"User inserted with ID: {userId}");
        
        // Query data
        Console.WriteLine("Querying user data...");
        var user = await GetUserAsync(userId);
        Console.WriteLine($"Retrieved user: {user.Name} - {user.Email}");
        
        // Batch operations
        Console.WriteLine("Performing batch operations...");
        await BatchOperationsAsync();
    }
    
    static async Task<int> InsertUserAsync(string name, string email)
    {
        // Simulate database insert operation
        await Task.Delay(500);
        return new Random().Next(1000, 9999);
    }
    
    static async Task<User> GetUserAsync(int id)
    {
        // Simulate database query operation
        await Task.Delay(300);
        return new User { Id = id, Name = "John Doe", Email = "john@example.com" };
    }
    
    static async Task BatchOperationsAsync()
    {
        var tasks = new List<Task<int>>();
        
        for (int i = 1; i <= 5; i++)
        {
            tasks.Add(InsertUserAsync($"User {i}", $"user{i}@example.com"));
        }
        
        var results = await Task.WhenAll(tasks);
        Console.WriteLine($"Batch insert completed. Created {results.Length} users.");
    }
}

class User
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Email { get; set; }
}
```

**Expected Output:**
```
Database Operations Demo
Inserting user data...
User inserted with ID: 4567
Querying user data...
Retrieved user: John Doe - john@example.com
Performing batch operations...
Batch insert completed. Created 5 users.
```

### Scenario 4: UI Responsiveness Simulation

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("UI Responsiveness Demo");
        Console.WriteLine("Simulating a responsive UI application...");
        
        // Simulate UI thread continuing to work while async operation runs
        await ResponsiveUIDemo();
    }
    
    static async Task ResponsiveUIDemo()
    {
        Console.WriteLine("UI: Starting heavy operation...");
        
        // Start the heavy operation
        var heavyTask = HeavyOperationAsync();
        
        // UI continues to be responsive
        for (int i = 1; i <= 5; i++)
        {
            Console.WriteLine($"UI: Processing user input {i}...");
            await Task.Delay(200); // Simulate UI work
        }
        
        // Wait for heavy operation to complete
        string result = await heavyTask;
        Console.WriteLine($"UI: Heavy operation completed: {result}");
        Console.WriteLine("UI: Application remains responsive throughout!");
    }
    
    static async Task<string> HeavyOperationAsync()
    {
        Console.WriteLine("Heavy Operation: Starting...");
        await Task.Delay(2000); // Simulate heavy work
        Console.WriteLine("Heavy Operation: Completed!");
        return "Heavy operation result";
    }
}
```

**Expected Output:**
```

Simulating a responsive UI application...
UI: Starting heavy operation...
Heavy Operation: Starting...
UI: Processing user input 1...
UI: Processing user input 2...
UI: Processing user input 3...
UI: Processing user input 4...
UI: Processing user input 5...
Heavy Operation: Completed!
UI: Heavy operation completed: Heavy operation result
UI: Application remains responsive throughout!
```

## Best Practices

1. **Use async/await for I/O operations**: File, network, database calls
2. **Avoid async void**: Except for event handlers
3. **ConfigureAwait(false)**: In library code to prevent deadlocks
4. **Handle exceptions**: Use try-catch around await calls
5. **Use Task.WhenAll**: For parallel operations
6. **Consider cancellation**: For long-running operations
7. **Profile performance**: Measure actual benefits before applying everywhere
