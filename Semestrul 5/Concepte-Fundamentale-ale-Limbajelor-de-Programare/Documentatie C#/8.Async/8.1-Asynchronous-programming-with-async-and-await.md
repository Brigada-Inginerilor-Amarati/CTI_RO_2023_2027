# 8.1 Asynchronous Programming with async and await

## Description

Asynchronous programming in C# allows you to write code that doesn't block the main thread while waiting for long-running operations to complete. The `async` and `await` keywords are the foundation of asynchronous programming in C#, making it easy to write code that can perform multiple operations concurrently without blocking the UI or consuming unnecessary resources.

## Key Concepts

- **async**: Modifies a method to indicate it contains asynchronous operations
- **await**: Suspends execution until the awaited task completes
- **Task**: Represents an asynchronous operation that can return a value
- **Non-blocking**: Allows the calling thread to continue with other work

## Code Examples

### Basic async/await Example

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Starting async operation...");
        
        // Call async method
        string result = await GetDataAsync();
        
        Console.WriteLine($"Result: {result}");
        Console.WriteLine("Main method completed.");
    }
    
    static async Task<string> GetDataAsync()
    {
        Console.WriteLine("GetDataAsync started...");
        
        // Simulate async work (like API call, file I/O, etc.)
        await Task.Delay(2000); // Wait for 2 seconds
        
        Console.WriteLine("GetDataAsync completed.");
        return "Data retrieved successfully!";
    }
}
```

**Expected Output:**
```
Starting async operation...
GetDataAsync started...
GetDataAsync completed.
Result: Data retrieved successfully!
Main method completed.
```

### Multiple Async Operations

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Starting multiple async operations...");
        
        // Start multiple operations concurrently
        var task1 = GetDataAsync("Task 1", 1000);
        var task2 = GetDataAsync("Task 2", 1500);
        var task3 = GetDataAsync("Task 3", 800);
        
        // Wait for all to complete
        var results = await Task.WhenAll(task1, task2, task3);
        
        Console.WriteLine("All operations completed:");
        foreach (var result in results)
        {
            Console.WriteLine($"- {result}");
        }
    }
    
    static async Task<string> GetDataAsync(string taskName, int delayMs)
    {
        Console.WriteLine($"{taskName} started...");
        await Task.Delay(delayMs);
        Console.WriteLine($"{taskName} completed.");
        return $"{taskName} result";
    }
}
```

**Expected Output:**
```
Starting multiple async operations...
Task 1 started...
Task 2 started...
Task 3 started...
Task 3 completed.
Task 1 completed.
Task 2 completed.
All operations completed:
- Task 1 result
- Task 2 result
- Task 3 result
```

### Async Method with Return Value

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Calculating sum asynchronously...");
        
        int result = await CalculateSumAsync(10, 20);
        
        Console.WriteLine($"Sum: {result}");
    }
    
    static async Task<int> CalculateSumAsync(int a, int b)
    {
        Console.WriteLine("Starting calculation...");
        
        // Simulate some processing time
        await Task.Delay(1000);
        
        int sum = a + b;
        Console.WriteLine("Calculation completed.");
        
        return sum;
    }
}
```

**Expected Output:**
```
Calculating sum asynchronously...
Starting calculation...
Calculation completed.
Sum: 30
```

## Important Notes

1. **async void**: Only use for event handlers. For all other cases, use `async Task` or `async Task<T>`
2. **ConfigureAwait(false)**: Use in library code to avoid deadlocks: `await SomeMethod().ConfigureAwait(false)`
3. **Exception Handling**: Use try-catch blocks around await calls to handle exceptions
4. **Performance**: Async methods have a small overhead, so only use them for I/O-bound operations or when you need to free up the thread

## Common Patterns

### Fire-and-Forget
```csharp
// Start operation but don't wait for it
_ = SomeAsyncOperation(); // Discard the task
```

### Exception Handling
```csharp
try
{
    await SomeAsyncOperation();
}
catch (Exception ex)
{
    Console.WriteLine($"Error: {ex.Message}");
}
```

### Timeout
```csharp
using (var cts = new CancellationTokenSource(TimeSpan.FromSeconds(5)))
{
    try
    {
        await SomeAsyncOperation(cts.Token);
    }
    catch (OperationCanceledException)
    {
        Console.WriteLine("Operation timed out.");
    }
}
```
