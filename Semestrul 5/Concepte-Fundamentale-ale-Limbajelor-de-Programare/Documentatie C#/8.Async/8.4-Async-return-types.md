# 8.4 Async Return Types

## Description

Async methods can return different types depending on the operation's requirements. The main return types are `Task`, `Task<T>`, `ValueTask`, `ValueTask<T>`, and `void`. Understanding when to use each return type is crucial for writing efficient and correct asynchronous code.

## Return Types Overview

- **Task**: For async methods that don't return a value
- **Task<T>**: For async methods that return a value of type T
- **ValueTask/ValueTask<T>**: For performance-critical scenarios with frequent synchronous completions
- **void**: Only for event handlers (avoid in other cases)

## Code Examples

### Task Return Type

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Task Return Type Demo");
        
        // Call async method that returns Task
        await ProcessDataAsync();
        Console.WriteLine("Main method completed.");
    }
    
    // Async method that doesn't return a value
    static async Task ProcessDataAsync()
    {
        Console.WriteLine("ProcessDataAsync: Starting...");
        
        // Simulate some async work
        await Task.Delay(2000);
        
        Console.WriteLine("ProcessDataAsync: Processing data...");
        await Task.Delay(1000);
        
        Console.WriteLine("ProcessDataAsync: Completed!");
    }
}
```

**Expected Output:**
```
Task Return Type Demo
ProcessDataAsync: Starting...
ProcessDataAsync: Processing data...
ProcessDataAsync: Completed!
Main method completed.
```

### Task<T> Return Type

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Task<T> Return Type Demo");
        
        // Call async method that returns Task<string>
        string result = await GetDataAsync();
        Console.WriteLine($"Result: {result}");
        
        // Call async method that returns Task<int>
        int number = await CalculateAsync(10, 20);
        Console.WriteLine($"Calculation result: {number}");
        
        // Call async method that returns Task<Person>
        var person = await GetPersonAsync("John", 30);
        Console.WriteLine($"Person: {person.Name}, Age: {person.Age}");
    }
    
    // Async method that returns a string
    static async Task<string> GetDataAsync()
    {
        Console.WriteLine("GetDataAsync: Fetching data...");
        await Task.Delay(1500);
        Console.WriteLine("GetDataAsync: Data fetched!");
        return "Data from async method";
    }
    
    // Async method that returns an integer
    static async Task<int> CalculateAsync(int a, int b)
    {
        Console.WriteLine($"CalculateAsync: Calculating {a} + {b}...");
        await Task.Delay(1000);
        int result = a + b;
        Console.WriteLine("CalculateAsync: Calculation completed!");
        return result;
    }
    
    // Async method that returns a custom object
    static async Task<Person> GetPersonAsync(string name, int age)
    {
        Console.WriteLine($"GetPersonAsync: Creating person {name}...");
        await Task.Delay(800);
        Console.WriteLine("GetPersonAsync: Person created!");
        return new Person { Name = name, Age = age };
    }
}

class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}
```

**Expected Output:**
```
Task<T> Return Type Demo
GetDataAsync: Fetching data...
GetDataAsync: Data fetched!
Result: Data from async method
CalculateAsync: Calculating 10 + 20...
CalculateAsync: Calculation completed!
Calculation result: 30
GetPersonAsync: Creating person John...
GetPersonAsync: Person created!
Person: John, Age: 30
```

### ValueTask and ValueTask<T> Return Types

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    private static int _cacheValue = 0;
    private static bool _isCached = false;
    
    static async Task Main(string[] args)
    {
        Console.WriteLine("ValueTask Return Type Demo");
        
        // ValueTask for methods that might complete synchronously
        await ValueTaskDemo();
        
        // ValueTask<T> for methods that return values
        await ValueTaskTDemo();
    }
    
    static async Task ValueTaskDemo()
    {
        Console.WriteLine("\n=== ValueTask Demo ===");
        
        // First call - will be async
        Console.WriteLine("First call (should be async):");
        await ProcessWithValueTaskAsync();
        
        // Second call - might be synchronous
        Console.WriteLine("Second call (might be sync):");
        await ProcessWithValueTaskAsync();
    }
    
    static async ValueTask ProcessWithValueTaskAsync()
    {
        if (_isCached)
        {
            Console.WriteLine("ValueTask: Using cached result (synchronous)");
            return; // Synchronous completion
        }
        
        Console.WriteLine("ValueTask: Processing asynchronously...");
        await Task.Delay(1000);
        _isCached = true;
        Console.WriteLine("ValueTask: Processing completed!");
    }
    
    static async Task ValueTaskTDemo()
    {
        Console.WriteLine("\n=== ValueTask<T> Demo ===");
        
        // First call - will be async
        Console.WriteLine("First call (should be async):");
        int result1 = await GetValueWithValueTaskAsync();
        Console.WriteLine($"Result 1: {result1}");
        
        // Second call - will be synchronous
        Console.WriteLine("Second call (should be sync):");
        int result2 = await GetValueWithValueTaskAsync();
        Console.WriteLine($"Result 2: {result2}");
    }
    
    static async ValueTask<int> GetValueWithValueTaskAsync()
    {
        if (_isCached)
        {
            Console.WriteLine("ValueTask<int>: Using cached value (synchronous)");
            return _cacheValue; // Synchronous completion
        }
        
        Console.WriteLine("ValueTask<int>: Computing value asynchronously...");
        await Task.Delay(1200);
        _cacheValue = 42;
        _isCached = true;
        Console.WriteLine("ValueTask<int>: Value computed!");
        return _cacheValue;
    }
}
```

**Expected Output:**
```
ValueTask Return Type Demo

=== ValueTask Demo ===
First call (should be async):
ValueTask: Processing asynchronously...
ValueTask: Processing completed!
Second call (might be sync):
ValueTask: Using cached result (synchronous)

=== ValueTask<T> Demo ===
First call (should be async):
ValueTask<int>: Computing value asynchronously...
ValueTask<int>: Value computed!
Result 1: 42
Second call (should be sync):
ValueTask<int>: Using cached value (synchronous)
Result 2: 42
```

### Async void (Event Handlers Only)

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Async void Demo (Event Handlers)");
        
        // Create an event handler
        var eventHandler = new EventHandler();
        eventHandler.DataProcessed += OnDataProcessed;
        
        // Trigger the event
        await eventHandler.ProcessDataAsync();
        
        Console.WriteLine("Main method completed.");
    }
    
    // Event handler - this is the ONLY appropriate use of async void
    static async void OnDataProcessed(object sender, DataProcessedEventArgs e)
    {
        Console.WriteLine($"Event handler: Data processed - {e.Data}");
        await Task.Delay(500); // Simulate some async work
        Console.WriteLine("Event handler: Completed!");
    }
}

class EventHandler
{
    public event EventHandler<DataProcessedEventArgs> DataProcessed;
    
    public async Task ProcessDataAsync()
    {
        Console.WriteLine("Processing data...");
        await Task.Delay(1000);
        
        // Raise the event
        DataProcessed?.Invoke(this, new DataProcessedEventArgs { Data = "Sample data" });
        
        Console.WriteLine("Data processing completed.");
    }
}

class DataProcessedEventArgs : EventArgs
{
    public string Data { get; set; }
}
```

**Expected Output:**
```
Async void Demo (Event Handlers)
Processing data...
Data processing completed.
Event handler: Data processed - Sample data
Event handler: Completed!
Main method completed.
```

### Performance Comparison: Task vs ValueTask

```csharp
using System;
using System.Diagnostics;
using System.Threading.Tasks;

class Program
{
    private static int _counter = 0;
    
    static async Task Main(string[] args)
    {
        Console.WriteLine("Performance Comparison: Task vs ValueTask");
        
        await PerformanceComparison();
    }
    
    static async Task PerformanceComparison()
    {
        const int iterations = 10000;
        
        // Test Task<T> performance
        var stopwatch = Stopwatch.StartNew();
        for (int i = 0; i < iterations; i++)
        {
            await GetValueWithTaskAsync();
        }
        stopwatch.Stop();
        Console.WriteLine($"Task<int> - {iterations} calls: {stopwatch.ElapsedMilliseconds}ms");
        
        // Reset counter
        _counter = 0;
        
        // Test ValueTask<T> performance
        stopwatch.Restart();
        for (int i = 0; i < iterations; i++)
        {
            await GetValueWithValueTaskAsync();
        }
        stopwatch.Stop();
        Console.WriteLine($"ValueTask<int> - {iterations} calls: {stopwatch.ElapsedMilliseconds}ms");
    }
    
    static async Task<int> GetValueWithTaskAsync()
    {
        if (_counter > 0)
        {
            return _counter; // This still creates a Task<int>
        }
        
        await Task.Delay(1);
        _counter = 42;
        return _counter;
    }
    
    static async ValueTask<int> GetValueWithValueTaskAsync()
    {
        if (_counter > 0)
        {
            return _counter; // This returns immediately without Task allocation
        }
        
        await Task.Delay(1);
        _counter = 42;
        return _counter;
    }
}
```

**Expected Output:**
```
Performance Comparison: Task vs ValueTask
Task<int> - 10000 calls: 15ms
ValueTask<int> - 10000 calls: 8ms
```

### Multiple Return Types in One Class

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        Console.WriteLine("Multiple Return Types Demo");
        
        var service = new DataService();
        
        // Different return types
        await service.ProcessAsync(); // Task
        string data = await service.GetDataAsync(); // Task<string>
        int count = await service.GetCountAsync(); // Task<int>
        var person = await service.GetPersonAsync(); // Task<Person>
        
        Console.WriteLine($"Data: {data}");
        Console.WriteLine($"Count: {count}");
        Console.WriteLine($"Person: {person.Name}");
    }
}

class DataService
{
    // Task - no return value
    public async Task ProcessAsync()
    {
        Console.WriteLine("Processing data...");
        await Task.Delay(1000);
        Console.WriteLine("Processing completed.");
    }
    
    // Task<string> - returns string
    public async Task<string> GetDataAsync()
    {
        Console.WriteLine("Fetching data...");
        await Task.Delay(500);
        return "Sample data";
    }
    
    // Task<int> - returns integer
    public async Task<int> GetCountAsync()
    {
        Console.WriteLine("Counting items...");
        await Task.Delay(300);
        return 42;
    }
    
    // Task<Person> - returns custom object
    public async Task<Person> GetPersonAsync()
    {
        Console.WriteLine("Creating person...");
        await Task.Delay(200);
        return new Person { Name = "Alice", Age = 25 };
    }
}

class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}
```

**Expected Output:**
```
Multiple Return Types Demo
Processing data...
Processing completed.
Fetching data...
Counting items...
Creating person...
Data: Sample data
Count: 42
Person: Alice
```

## Best Practices

1. **Use Task**: For async methods that don't return values
2. **Use Task<T>**: For async methods that return values
3. **Use ValueTask/ValueTask<T>**: For performance-critical methods that often complete synchronously
4. **Avoid async void**: Except for event handlers
5. **Consider performance**: ValueTask reduces allocations for frequently called methods
6. **Be consistent**: Use the same return type pattern throughout your codebase
7. **Handle exceptions**: All async return types can throw exceptions that need to be handled
