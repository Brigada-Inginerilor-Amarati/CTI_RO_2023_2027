# 1.2 Program Structure

## Description

A C# program consists of one or more source files that contain type declarations. The fundamental structure of a C# program includes namespaces, classes, methods, and statements. Understanding this hierarchy is essential for organizing code effectively.

**Key Components:**
- **Namespaces**: Organize code and prevent naming conflicts
- **Classes**: Define types and contain members (fields, properties, methods)
- **Methods**: Contain executable statements
- **Statements**: Perform actions (assignments, method calls, loops, etc.)

**Typical Structure:**
```
Namespace
  └── Class
       └── Method
            └── Statements
```

## Code Examples

### Basic Program Structure

```csharp
// Using directives - import namespaces
using System;
using System.Collections.Generic;

// Namespace declaration
namespace MyApplication
{
    // Class declaration
    class Program
    {
        // Method declaration
        static void Main(string[] args)
        {
            // Statements
            Console.WriteLine("This demonstrates C# program structure");
        }
    }
}
```

**Output:**
```
This demonstrates C# program structure
```

### Multiple Classes in One Namespace

```csharp
using System;

namespace School
{
    // First class
    class Student
    {
        public string Name { get; set; }
        public int Grade { get; set; }

        public void DisplayInfo()
        {
            Console.WriteLine($"Student: {Name}, Grade: {Grade}");
        }
    }

    // Second class
    class Teacher
    {
        public string Name { get; set; }
        public string Subject { get; set; }

        public void DisplayInfo()
        {
            Console.WriteLine($"Teacher: {Name}, Subject: {Subject}");
        }
    }

    // Main program class
    class Program
    {
        static void Main()
        {
            Student student = new Student
            {
                Name = "Alice",
                Grade = 10
            };

            Teacher teacher = new Teacher
            {
                Name = "Mr. Smith",
                Subject = "Mathematics"
            };

            student.DisplayInfo();
            teacher.DisplayInfo();
        }
    }
}
```

**Output:**
```
Student: Alice, Grade: 10
Teacher: Mr. Smith, Subject: Mathematics
```

### Nested Namespaces

```csharp
using System;

namespace Company
{
    namespace Employees
    {
        class Manager
        {
            public string Name { get; set; }

            public void Work()
            {
                Console.WriteLine($"{Name} is managing the team");
            }
        }
    }

    namespace Products
    {
        class Software
        {
            public string Name { get; set; }

            public void Display()
            {
                Console.WriteLine($"Product: {Name}");
            }
        }
    }
}

namespace Company
{
    class Program
    {
        static void Main()
        {
            // Access nested namespace classes
            Employees.Manager manager = new Employees.Manager
            {
                Name = "Sarah"
            };

            Products.Software product = new Products.Software
            {
                Name = "CloudApp"
            };

            manager.Work();
            product.Display();
        }
    }
}
```

**Output:**
```
Sarah is managing the team
Product: CloudApp
```

### Using Multiple Namespaces

```csharp
using System;
using System.Text;
using System.Collections.Generic;

namespace DataProcessing
{
    class Program
    {
        static void Main()
        {
            // Using System namespace
            Console.WriteLine("Processing data...");

            // Using System.Text namespace
            StringBuilder sb = new StringBuilder();
            sb.Append("Hello");
            sb.Append(" ");
            sb.Append("World");
            Console.WriteLine(sb.ToString());

            // Using System.Collections.Generic namespace
            List<int> numbers = new List<int> { 1, 2, 3, 4, 5 };
            Console.WriteLine($"Count: {numbers.Count}");
        }
    }
}
```

**Output:**
```
Processing data...
Hello World
Count: 5
```

### Class Members - Fields, Properties, and Methods

```csharp
using System;

namespace BankingSystem
{
    class BankAccount
    {
        // Field (private)
        private decimal balance;

        // Property (public)
        public string AccountNumber { get; set; }
        public string Owner { get; set; }

        // Constructor
        public BankAccount(string accountNumber, string owner, decimal initialBalance)
        {
            AccountNumber = accountNumber;
            Owner = owner;
            balance = initialBalance;
        }

        // Methods
        public void Deposit(decimal amount)
        {
            balance += amount;
            Console.WriteLine($"Deposited: ${amount}");
        }

        public void Withdraw(decimal amount)
        {
            if (amount <= balance)
            {
                balance -= amount;
                Console.WriteLine($"Withdrew: ${amount}");
            }
            else
            {
                Console.WriteLine("Insufficient funds!");
            }
        }

        public void DisplayBalance()
        {
            Console.WriteLine($"Account: {AccountNumber}");
            Console.WriteLine($"Owner: {Owner}");
            Console.WriteLine($"Balance: ${balance}");
        }
    }

    class Program
    {
        static void Main()
        {
            BankAccount account = new BankAccount("12345", "John Doe", 1000m);

            account.DisplayBalance();
            Console.WriteLine();

            account.Deposit(500m);
            account.Withdraw(200m);
            Console.WriteLine();

            account.DisplayBalance();
        }
    }
}
```

**Output:**
```
Account: 12345
Owner: John Doe
Balance: $1000

Deposited: $500
Withdrew: $200

Account: 12345
Owner: John Doe
Balance: $1300
```

### Multiple Files Structure (Conceptual)

```csharp
// File: Models/Person.cs
using System;

namespace MyApp.Models
{
    public class Person
    {
        public string FirstName { get; set; }
        public string LastName { get; set; }

        public string GetFullName()
        {
            return $"{FirstName} {LastName}";
        }
    }
}

// File: Services/PersonService.cs
using System;
using MyApp.Models;

namespace MyApp.Services
{
    public class PersonService
    {
        public void PrintPerson(Person person)
        {
            Console.WriteLine($"Person: {person.GetFullName()}");
        }
    }
}

// File: Program.cs
using System;
using MyApp.Models;
using MyApp.Services;

namespace MyApp
{
    class Program
    {
        static void Main()
        {
            Person person = new Person
            {
                FirstName = "Jane",
                LastName = "Smith"
            };

            PersonService service = new PersonService();
            service.PrintPerson(person);
        }
    }
}
```

**Output:**
```
Person: Jane Smith
```

## Structure Best Practices

1. **One namespace per logical grouping** - Group related classes together
2. **One public class per file** - Makes code easier to find and maintain
3. **Meaningful names** - Use descriptive names for namespaces, classes, and methods
4. **Consistent organization** - Follow a consistent folder and namespace structure
5. **Access modifiers** - Use appropriate access levels (public, private, protected, internal)
