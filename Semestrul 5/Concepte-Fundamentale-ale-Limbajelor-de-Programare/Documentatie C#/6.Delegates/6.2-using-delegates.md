# 6.2 Using Delegates

## Description

This section demonstrates practical applications of delegates in real-world scenarios. Delegates are commonly used for callbacks, event handling, method parameters, and implementing flexible architectures.

### Common Use Cases:
- **Callback mechanisms**: Notify when operations complete
- **Event handling**: Respond to user interactions
- **Plugin systems**: Allow external code to extend functionality
- **Sorting and filtering**: Pass comparison logic to algorithms
- **Asynchronous programming**: Handle completion callbacks

## Code Examples

### Example 1: Callback Pattern with File Operations

```csharp
using System;
using System.IO;

// Delegate for file operation callbacks
public delegate void FileOperationCallback(bool success, string message);

class FileManager
{
    public static void ReadFileAsync(string filePath, FileOperationCallback callback)
    {
        try
        {
            if (File.Exists(filePath))
            {
                string content = File.ReadAllText(filePath);
                callback(true, $"File read successfully. Content length: {content.Length} characters");
            }
            else
            {
                callback(false, "File not found");
            }
        }
        catch (Exception ex)
        {
            callback(false, $"Error reading file: {ex.Message}");
        }
    }

    public static void WriteFileAsync(string filePath, string content, FileOperationCallback callback)
    {
        try
        {
            File.WriteAllText(filePath, content);
            callback(true, "File written successfully");
        }
        catch (Exception ex)
        {
            callback(false, $"Error writing file: {ex.Message}");
        }
    }
}

class Program
{
    static void OnFileOperationComplete(bool success, string message)
    {
        if (success)
        {
            Console.WriteLine($"✓ Success: {message}");
        }
        else
        {
            Console.WriteLine($"✗ Error: {message}");
        }
    }

    static void Main()
    {
        string filePath = "test.txt";
        string content = "Hello, World! This is a test file.";

        // Write file with callback
        FileManager.WriteFileAsync(filePath, content, OnFileOperationComplete);

        // Read file with callback
        FileManager.ReadFileAsync(filePath, OnFileOperationComplete);

        // Try to read non-existent file
        FileManager.ReadFileAsync("nonexistent.txt", OnFileOperationComplete);
    }
}
```

**Output:**
```
✓ Success: File written successfully
✓ Success: File read successfully. Content length: 37 characters
✗ Error: File not found
```

### Example 2: Sorting with Custom Comparison

```csharp
using System;
using System.Collections.Generic;

// Delegate for comparison operations
public delegate int CompareDelegate<T>(T x, T y);

class CustomSorter
{
    public static void BubbleSort<T>(T[] array, CompareDelegate<T> compare)
    {
        int n = array.Length;
        for (int i = 0; i < n - 1; i++)
        {
            for (int j = 0; j < n - i - 1; j++)
            {
                if (compare(array[j], array[j + 1]) > 0)
                {
                    // Swap elements
                    T temp = array[j];
                    array[j] = array[j + 1];
                    array[j + 1] = temp;
                }
            }
        }
    }
}

class Program
{
    // Comparison methods
    static int CompareIntegers(int x, int y)
    {
        return x.CompareTo(y);
    }

    static int CompareStrings(string x, string y)
    {
        return string.Compare(x, y, StringComparison.OrdinalIgnoreCase);
    }

    static int CompareByLength(string x, string y)
    {
        return x.Length.CompareTo(y.Length);
    }

    static void Main()
    {
        // Sort integers
        int[] numbers = { 64, 34, 25, 12, 22, 11, 90 };
        Console.WriteLine("Original numbers: " + string.Join(", ", numbers));
        
        CustomSorter.BubbleSort(numbers, CompareIntegers);
        Console.WriteLine("Sorted numbers: " + string.Join(", ", numbers));

        Console.WriteLine();

        // Sort strings alphabetically
        string[] names = { "Charlie", "Alice", "Bob", "David" };
        Console.WriteLine("Original names: " + string.Join(", ", names));
        
        CustomSorter.BubbleSort(names, CompareStrings);
        Console.WriteLine("Sorted alphabetically: " + string.Join(", ", names));

        Console.WriteLine();

        // Sort strings by length
        string[] words = { "apple", "pie", "banana", "cat" };
        Console.WriteLine("Original words: " + string.Join(", ", words));
        
        CustomSorter.BubbleSort(words, CompareByLength);
        Console.WriteLine("Sorted by length: " + string.Join(", ", words));
    }
}
```

**Output:**
```
Original numbers: 64, 34, 25, 12, 22, 11, 90
Sorted numbers: 11, 12, 22, 25, 34, 64, 90

Original names: Charlie, Alice, Bob, David
Sorted alphabetically: Alice, Bob, Charlie, David

Original words: apple, pie, banana, cat
Sorted by length: pie, cat, apple, banana
```

### Example 3: Plugin Architecture

```csharp
using System;
using System.Collections.Generic;

// Delegate for processing data
public delegate string DataProcessor(string input);

// Plugin interface
public interface IPlugin
{
    string Name { get; }
    string Process(string input);
}

// Text processing plugins
class UppercasePlugin : IPlugin
{
    public string Name => "Uppercase";
    public string Process(string input) => input.ToUpper();
}

class LowercasePlugin : IPlugin
{
    public string Name => "Lowercase";
    public string Process(string input) => input.ToLower();
}

class ReversePlugin : IPlugin
{
    public string Name => "Reverse";
    public string Process(string input)
    {
        char[] chars = input.ToCharArray();
        Array.Reverse(chars);
        return new string(chars);
    }
}

class PluginManager
{
    private List<IPlugin> plugins = new List<IPlugin>();

    public void RegisterPlugin(IPlugin plugin)
    {
        plugins.Add(plugin);
        Console.WriteLine($"Registered plugin: {plugin.Name}");
    }

    public void ProcessWithAllPlugins(string input)
    {
        Console.WriteLine($"\nProcessing: '{input}'");
        Console.WriteLine("Results:");
        
        foreach (var plugin in plugins)
        {
            string result = plugin.Process(input);
            Console.WriteLine($"  {plugin.Name}: {result}");
        }
    }

    public void ProcessWithDelegate(string input, DataProcessor processor)
    {
        string result = processor(input);
        Console.WriteLine($"Delegate result: {result}");
    }
}

class Program
{
    static void Main()
    {
        PluginManager manager = new PluginManager();

        // Register plugins
        manager.RegisterPlugin(new UppercasePlugin());
        manager.RegisterPlugin(new LowercasePlugin());
        manager.RegisterPlugin(new ReversePlugin());

        string text = "Hello World";

        // Process with all plugins
        manager.ProcessWithAllPlugins(text);

        Console.WriteLine("\n" + new string('-', 40));

        // Use delegates for custom processing
        DataProcessor customProcessor = (input) => $"Custom: {input} - Length: {input.Length}";
        manager.ProcessWithDelegate(text, customProcessor);

        // Chain multiple processors
        DataProcessor chainedProcessor = (input) =>
        {
            string upper = input.ToUpper();
            char[] chars = upper.ToCharArray();
            Array.Reverse(chars);
            return new string(chars);
        };
        manager.ProcessWithDelegate(text, chainedProcessor);
    }
}
```

**Output:**
```
Registered plugin: Uppercase
Registered plugin: Lowercase
Registered plugin: Reverse

Processing: 'Hello World'
Results:
  Uppercase: HELLO WORLD
  Lowercase: hello world
  Reverse: dlroW olleH

----------------------------------------
Delegate result: Custom: Hello World - Length: 11
Delegate result: DLROW OLLEH
```

### Example 4: Event Simulation with Delegates

```csharp
using System;

// Delegate for event handling
public delegate void EventHandler(string eventName, object data);

class EventSimulator
{
    private EventHandler eventHandler;

    public void Subscribe(EventHandler handler)
    {
        eventHandler += handler;
        Console.WriteLine("Event handler subscribed");
    }

    public void Unsubscribe(EventHandler handler)
    {
        eventHandler -= handler;
        Console.WriteLine("Event handler unsubscribed");
    }

    public void TriggerEvent(string eventName, object data)
    {
        Console.WriteLine($"\nTriggering event: {eventName}");
        eventHandler?.Invoke(eventName, data);
    }
}

class Program
{
    static void OnUserLogin(string eventName, object data)
    {
        Console.WriteLine($"  [LOGIN] User logged in: {data}");
    }

    static void OnUserLogout(string eventName, object data)
    {
        Console.WriteLine($"  [LOGOUT] User logged out: {data}");
    }

    static void OnDataChanged(string eventName, object data)
    {
        Console.WriteLine($"  [DATA] Data changed: {data}");
    }

    static void Main()
    {
        EventSimulator simulator = new EventSimulator();

        // Subscribe to events
        simulator.Subscribe(OnUserLogin);
        simulator.Subscribe(OnUserLogout);
        simulator.Subscribe(OnDataChanged);

        // Trigger events
        simulator.TriggerEvent("UserLogin", "Alice");
        simulator.TriggerEvent("UserLogout", "Bob");
        simulator.TriggerEvent("DataChanged", "User profile updated");

        Console.WriteLine("\n" + new string('-', 30));

        // Unsubscribe one handler
        simulator.Unsubscribe(OnDataChanged);

        // Trigger events again
        simulator.TriggerEvent("UserLogin", "Charlie");
        simulator.TriggerEvent("DataChanged", "This won't be handled");
    }
}
```

**Output:**
```
Event handler subscribed
Event handler subscribed
Event handler subscribed

Triggering event: UserLogin
  [LOGIN] User logged in: Alice
  [LOGOUT] User logged in: Alice
  [DATA] User logged in: Alice

Triggering event: UserLogout
  [LOGIN] User logged out: Bob
  [LOGOUT] User logged out: Bob
  [DATA] User logged out: Bob

Triggering event: DataChanged
  [LOGIN] Data changed: User profile updated
  [LOGOUT] Data changed: User profile updated
  [DATA] Data changed: User profile updated

------------------------------
Event handler unsubscribed

Triggering event: UserLogin
  [LOGIN] User logged in: Charlie
  [LOGOUT] User logged in: Charlie

Triggering event: DataChanged
  [LOGIN] Data changed: This won't be handled
  [LOGOUT] Data changed: This won't be handled
```

### Example 5: Validation Framework

```csharp
using System;
using System.Collections.Generic;

// Delegate for validation rules
public delegate bool ValidationRule<T>(T value, out string errorMessage);

class Validator<T>
{
    private List<ValidationRule<T>> rules = new List<ValidationRule<T>>();

    public void AddRule(ValidationRule<T> rule)
    {
        rules.Add(rule);
    }

    public bool Validate(T value, out List<string> errors)
    {
        errors = new List<string>();
        bool isValid = true;

        foreach (var rule in rules)
        {
            if (!rule(value, out string errorMessage))
            {
                errors.Add(errorMessage);
                isValid = false;
            }
        }

        return isValid;
    }
}

class Program
{
    static void Main()
    {
        // Create validator for email
        var emailValidator = new Validator<string>();
        
        // Add validation rules
        emailValidator.AddRule((email, out string error) =>
        {
            if (string.IsNullOrEmpty(email))
            {
                error = "Email cannot be empty";
                return false;
            }
            error = null;
            return true;
        });

        emailValidator.AddRule((email, out string error) =>
        {
            if (!email.Contains("@"))
            {
                error = "Email must contain @ symbol";
                return false;
            }
            error = null;
            return true;
        });

        emailValidator.AddRule((email, out string error) =>
        {
            if (email.Length < 5)
            {
                error = "Email must be at least 5 characters";
                return false;
            }
            error = null;
            return true;
        });

        // Test validation
        string[] testEmails = { "", "invalid", "test@", "valid@email.com" };

        foreach (string email in testEmails)
        {
            Console.WriteLine($"\nValidating: '{email}'");
            
            if (emailValidator.Validate(email, out List<string> errors))
            {
                Console.WriteLine("✓ Valid email");
            }
            else
            {
                Console.WriteLine("✗ Invalid email:");
                foreach (string error in errors)
                {
                    Console.WriteLine($"  - {error}");
                }
            }
        }
    }
}
```

**Output:**
```
Validating: ''
✗ Invalid email:
  - Email cannot be empty
  - Email must contain @ symbol
  - Email must be at least 5 characters

Validating: 'invalid'
✗ Invalid email:
  - Email must contain @ symbol
  - Email must be at least 5 characters

Validating: 'test@'
✗ Invalid email:
  - Email must be at least 5 characters

Validating: 'valid@email.com'
✓ Valid email
```

## Key Points

- **Callback Pattern**: Delegates enable clean separation of concerns
- **Flexibility**: Same method can work with different delegate implementations
- **Plugin Architecture**: Easy to extend functionality without modifying existing code
- **Event Handling**: Foundation for event-driven programming
- **Validation**: Create reusable validation frameworks
- **Type Safety**: Compile-time checking ensures method signatures match
