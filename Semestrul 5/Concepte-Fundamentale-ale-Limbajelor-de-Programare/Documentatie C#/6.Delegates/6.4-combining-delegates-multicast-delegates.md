# 6.4 Combining Delegates. Multicast Delegates

## Description

**Multicast delegates** are delegates that can hold references to multiple methods. When a multicast delegate is invoked, all the methods it references are called in sequence. This is achieved by combining delegates using the `+` and `+=` operators.

### Key Concepts:
- **Multicast**: A single delegate can call multiple methods
- **Combining**: Use `+` or `+=` to add methods to a delegate
- **Removing**: Use `-` or `-=` to remove methods from a delegate
- **Invocation List**: The ordered list of methods that will be called
- **Return Values**: Only the last method's return value is returned

### When to Use:
- **Event handling**: Multiple handlers for the same event
- **Logging**: Multiple loggers for different outputs
- **Validation**: Multiple validation rules
- **Processing pipelines**: Sequential data processing

## Code Examples

### Example 1: Basic Multicast Delegate Operations

```csharp
using System;

public delegate void SimpleDelegate(string message);

class Program
{
    static void Method1(string message)
    {
        Console.WriteLine($"Method1: {message}");
    }

    static void Method2(string message)
    {
        Console.WriteLine($"Method2: {message}");
    }

    static void Method3(string message)
    {
        Console.WriteLine($"Method3: {message}");
    }

    static void Main()
    {
        // Create individual delegates
        SimpleDelegate del1 = Method1;
        SimpleDelegate del2 = Method2;
        SimpleDelegate del3 = Method3;

        Console.WriteLine("Individual delegates:");
        del1("Hello");
        del2("World");
        del3("C#");

        Console.WriteLine("\n" + new string('-', 30));

        // Combine delegates using + operator
        SimpleDelegate combined = del1 + del2 + del3;
        Console.WriteLine("Combined delegate (+ operator):");
        combined("Multicast");

        Console.WriteLine("\n" + new string('-', 30));

        // Combine using += operator
        SimpleDelegate multicast = Method1;
        multicast += Method2;
        multicast += Method3;
        
        Console.WriteLine("Multicast delegate (+= operator):");
        multicast("Sequential");

        Console.WriteLine("\n" + new string('-', 30));

        // Remove methods using -= operator
        multicast -= Method2;
        Console.WriteLine("After removing Method2:");
        multicast("Reduced");
    }
}
```

**Output:**
```
Individual delegates:
Method1: Hello
Method2: World
Method3: C#

------------------------------
Combined delegate (+ operator):
Method1: Multicast
Method2: Multicast
Method3: Multicast

------------------------------
Multicast delegate (+= operator):
Method1: Sequential
Method2: Sequential
Method3: Sequential

------------------------------
After removing Method2:
Method1: Reduced
Method3: Reduced
```

### Example 2: Return Values in Multicast Delegates

```csharp
using System;

public delegate int MathDelegate(int x, int y);

class Program
{
    static int Add(int x, int y)
    {
        int result = x + y;
        Console.WriteLine($"Add: {x} + {y} = {result}");
        return result;
    }

    static int Multiply(int x, int y)
    {
        int result = x * y;
        Console.WriteLine($"Multiply: {x} * {y} = {result}");
        return result;
    }

    static int Subtract(int x, int y)
    {
        int result = x - y;
        Console.WriteLine($"Subtract: {x} - {y} = {result}");
        return result;
    }

    static void Main()
    {
        // Create multicast delegate
        MathDelegate mathOps = Add;
        mathOps += Multiply;
        mathOps += Subtract;

        Console.WriteLine("All methods will execute:");
        int result = mathOps(10, 5);
        Console.WriteLine($"Return value (last method): {result}");

        Console.WriteLine("\n" + new string('-', 40));

        // Demonstrate that all methods execute but only last return value is used
        MathDelegate singleOp = Add;
        Console.WriteLine("Single operation:");
        int singleResult = singleOp(10, 5);
        Console.WriteLine($"Return value: {singleResult}");
    }
}
```

**Output:**
```
All methods will execute:
Add: 10 + 5 = 15
Multiply: 10 * 5 = 50
Subtract: 10 - 5 = 5
Return value (last method): 5

----------------------------------------
Single operation:
Add: 10 + 5 = 15
Return value: 15
```

### Example 3: Event Simulation with Multicast

```csharp
using System;

public delegate void EventHandler(string eventName, object data);

class EventManager
{
    public event EventHandler EventOccurred;

    public void TriggerEvent(string eventName, object data)
    {
        Console.WriteLine($"\nTriggering event: {eventName}");
        EventOccurred?.Invoke(eventName, data);
    }
}

class Program
{
    static void LogHandler(string eventName, object data)
    {
        Console.WriteLine($"  [LOG] {eventName}: {data}");
    }

    static void EmailHandler(string eventName, object data)
    {
        Console.WriteLine($"  [EMAIL] Sending notification for {eventName}");
    }

    static void DatabaseHandler(string eventName, object data)
    {
        Console.WriteLine($"  [DB] Storing {eventName} event");
    }

    static void AlertHandler(string eventName, object data)
    {
        if (eventName.Contains("ERROR"))
        {
            Console.WriteLine($"  [ALERT] Critical error: {data}");
        }
    }

    static void Main()
    {
        EventManager manager = new EventManager();

        // Subscribe multiple handlers
        manager.EventOccurred += LogHandler;
        manager.EventOccurred += EmailHandler;
        manager.EventOccurred += DatabaseHandler;
        manager.EventOccurred += AlertHandler;

        // Trigger events
        manager.TriggerEvent("UserLogin", "Alice");
        manager.TriggerEvent("DataUpdate", "User profile");
        manager.TriggerEvent("ERROR_Database", "Connection failed");

        Console.WriteLine("\n" + new string('-', 50));

        // Unsubscribe some handlers
        manager.EventOccurred -= EmailHandler;
        manager.EventOccurred -= DatabaseHandler;

        Console.WriteLine("After unsubscribing email and database handlers:");
        manager.TriggerEvent("UserLogout", "Alice");
    }
}
```

**Output:**
```
Triggering event: UserLogin
  [LOG] UserLogin: Alice
  [EMAIL] Sending notification for UserLogin
  [DB] Storing UserLogin event

Triggering event: DataUpdate
  [LOG] DataUpdate: User profile
  [EMAIL] Sending notification for DataUpdate
  [DB] Storing DataUpdate event

Triggering event: ERROR_Database
  [LOG] ERROR_Database: Connection failed
  [EMAIL] Sending notification for ERROR_Database
  [DB] Storing ERROR_Database event
  [ALERT] Critical error: Connection failed

--------------------------------------------------

After unsubscribing email and database handlers:
Triggering event: UserLogout
  [LOG] UserLogout: Alice
  [ALERT] Critical error: Alice
```

### Example 4: Processing Pipeline with Multicast

```csharp
using System;
using System.Collections.Generic;

public delegate string TextProcessor(string input);

class TextPipeline
{
    private List<TextProcessor> processors = new List<TextProcessor>();

    public void AddProcessor(TextProcessor processor)
    {
        processors.Add(processor);
    }

    public void RemoveProcessor(TextProcessor processor)
    {
        processors.Remove(processor);
    }

    public string ProcessText(string input)
    {
        string result = input;
        Console.WriteLine($"Original: {result}");

        foreach (var processor in processors)
        {
            result = processor(result);
        }

        return result;
    }
}

class Program
{
    static string ToUpperCase(string input)
    {
        string result = input.ToUpper();
        Console.WriteLine($"  After ToUpper: {result}");
        return result;
    }

    static string AddPrefix(string input)
    {
        string result = "PROCESSED: " + input;
        Console.WriteLine($"  After AddPrefix: {result}");
        return result;
    }

    static string AddSuffix(string input)
    {
        string result = input + " [COMPLETE]";
        Console.WriteLine($"  After AddSuffix: {result}");
        return result;
    }

    static string RemoveSpaces(string input)
    {
        string result = input.Replace(" ", "");
        Console.WriteLine($"  After RemoveSpaces: {result}");
        return result;
    }

    static void Main()
    {
        TextPipeline pipeline = new TextPipeline();

        // Add processors
        pipeline.AddProcessor(ToUpperCase);
        pipeline.AddProcessor(AddPrefix);
        pipeline.AddProcessor(AddSuffix);

        string text = "hello world";
        Console.WriteLine("Processing pipeline:");
        string result = pipeline.ProcessText(text);
        Console.WriteLine($"Final result: {result}");

        Console.WriteLine("\n" + new string('-', 40));

        // Add another processor
        pipeline.AddProcessor(RemoveSpaces);
        Console.WriteLine("After adding RemoveSpaces processor:");
        result = pipeline.ProcessText(text);
        Console.WriteLine($"Final result: {result}");

        Console.WriteLine("\n" + new string('-', 40));

        // Remove a processor
        pipeline.RemoveProcessor(AddSuffix);
        Console.WriteLine("After removing AddSuffix processor:");
        result = pipeline.ProcessText(text);
        Console.WriteLine($"Final result: {result}");
    }
}
```

**Output:**
```
Processing pipeline:
Original: hello world
  After ToUpper: HELLO WORLD
  After AddPrefix: PROCESSED: HELLO WORLD
  After AddSuffix: PROCESSED: HELLO WORLD [COMPLETE]
Final result: PROCESSED: HELLO WORLD [COMPLETE]

----------------------------------------
After adding RemoveSpaces processor:
Original: hello world
  After ToUpper: HELLO WORLD
  After AddPrefix: PROCESSED: HELLO WORLD
  After AddSuffix: PROCESSED: HELLO WORLD [COMPLETE]
  After RemoveSpaces: PROCESSED:HELLOWORLD[COMPLETE]
Final result: PROCESSED:HELLOWORLD[COMPLETE]

----------------------------------------
After removing AddSuffix processor:
Original: hello world
  After ToUpper: HELLO WORLD
  After AddPrefix: PROCESSED: HELLO WORLD
  After RemoveSpaces: PROCESSED:HELLOWORLD
Final result: PROCESSED:HELLOWORLD
```

### Example 5: Error Handling in Multicast Delegates

```csharp
using System;

public delegate void ProcessDelegate(string data);

class SafeMulticast
{
    public static void SafeInvoke(ProcessDelegate multicastDelegate, string data)
    {
        if (multicastDelegate == null) return;

        // Get invocation list
        Delegate[] invocationList = multicastDelegate.GetInvocationList();
        
        foreach (ProcessDelegate singleDelegate in invocationList)
        {
            try
            {
                singleDelegate(data);
            }
            catch (Exception ex)
            {
                Console.WriteLine($"  Error in {singleDelegate.Method.Name}: {ex.Message}");
            }
        }
    }
}

class Program
{
    static void Process1(string data)
    {
        Console.WriteLine($"  Process1: {data}");
    }

    static void Process2(string data)
    {
        Console.WriteLine($"  Process2: {data}");
        if (data.Contains("error"))
        {
            throw new Exception("Simulated error in Process2");
        }
    }

    static void Process3(string data)
    {
        Console.WriteLine($"  Process3: {data}");
    }

    static void Process4(string data)
    {
        Console.WriteLine($"  Process4: {data}");
        if (data.Contains("fail"))
        {
            throw new InvalidOperationException("Simulated failure in Process4");
        }
    }

    static void Main()
    {
        // Create multicast delegate
        ProcessDelegate processor = Process1;
        processor += Process2;
        processor += Process3;
        processor += Process4;

        Console.WriteLine("Safe multicast invocation:");
        SafeMulticast.SafeInvoke(processor, "normal data");

        Console.WriteLine("\nWith error data:");
        SafeMulticast.SafeInvoke(processor, "data with error");

        Console.WriteLine("\nWith failure data:");
        SafeMulticast.SafeInvoke(processor, "data with fail");

        Console.WriteLine("\n" + new string('-', 40));

        // Demonstrate unsafe invocation
        Console.WriteLine("Unsafe multicast invocation (stops on first error):");
        try
        {
            processor("data with error");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"Exception caught: {ex.Message}");
        }
    }
}
```

**Output:**
```
Safe multicast invocation:
  Process1: normal data
  Process2: normal data
  Process3: normal data
  Process4: normal data

With error data:
  Process1: data with error
  Error in Process2: Simulated error in Process2
  Process3: data with error
  Process4: data with error

With failure data:
  Process1: data with fail
  Process2: data with fail
  Process3: data with fail
  Error in Process4: Simulated failure in Process4

----------------------------------------
Unsafe multicast invocation (stops on first error):
  Process1: data with error
  Exception caught: Simulated error in Process2
```

### Example 6: Delegate Invocation List Inspection

```csharp
using System;

public delegate void InfoDelegate(string message);

class Program
{
    static void LogInfo(string message)
    {
        Console.WriteLine($"[LOG] {message}");
    }

    static void DebugInfo(string message)
    {
        Console.WriteLine($"[DEBUG] {message}");
    }

    static void ErrorInfo(string message)
    {
        Console.WriteLine($"[ERROR] {message}");
    }

    static void Main()
    {
        // Create multicast delegate
        InfoDelegate infoHandler = LogInfo;
        infoHandler += DebugInfo;
        infoHandler += ErrorInfo;

        // Inspect invocation list
        Console.WriteLine("Invocation list inspection:");
        Delegate[] invocationList = infoHandler.GetInvocationList();
        
        Console.WriteLine($"Number of methods: {invocationList.Length}");
        
        for (int i = 0; i < invocationList.Length; i++)
        {
            InfoDelegate singleDelegate = (InfoDelegate)invocationList[i];
            Console.WriteLine($"  {i + 1}. {singleDelegate.Method.Name} in {singleDelegate.Method.DeclaringType.Name}");
        }

        Console.WriteLine("\n" + new string('-', 40));

        // Invoke all methods
        Console.WriteLine("Invoking all methods:");
        infoHandler("Test message");

        Console.WriteLine("\n" + new string('-', 40));

        // Remove middle method
        infoHandler -= DebugInfo;
        Console.WriteLine("After removing DebugInfo:");
        infoHandler("Test message after removal");

        Console.WriteLine("\n" + new string('-', 40));

        // Check if delegate is null
        InfoDelegate emptyDelegate = null;
        Console.WriteLine($"Empty delegate is null: {emptyDelegate == null}");
        
        // Safe invocation
        emptyDelegate?.Invoke("This won't be called");
        Console.WriteLine("Safe invocation of null delegate completed");
    }
}
```

**Output:**
```
Invocation list inspection:
Number of methods: 3
  1. LogInfo in Program
  2. DebugInfo in Program
  3. ErrorInfo in Program

----------------------------------------
Invoking all methods:
[LOG] Test message
[DEBUG] Test message
[ERROR] Test message

----------------------------------------
After removing DebugInfo:
[LOG] Test message after removal
[ERROR] Test message after removal

----------------------------------------
Empty delegate is null: True
Safe invocation of null delegate completed
```

## Key Points

### Multicast Delegate Characteristics
- **Sequential Execution**: Methods are called in the order they were added
- **Return Values**: Only the last method's return value is returned
- **Exception Handling**: If one method throws an exception, subsequent methods may not execute
- **Null Safety**: Always check for null before invoking

### Operations
- **Addition**: Use `+` or `+=` to add methods
- **Removal**: Use `-` or `-=` to remove methods
- **Inspection**: Use `GetInvocationList()` to examine the methods
- **Safe Invocation**: Implement error handling for robust multicast delegates

### Best Practices
- **Error Handling**: Implement safe invocation patterns
- **Order Matters**: Be aware of execution order
- **Memory Management**: Remove delegates when no longer needed
- **Null Checks**: Always check for null before invocation
- **Return Values**: Don't rely on return values from multicast delegates
